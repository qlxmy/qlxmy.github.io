deepin安装Android模拟器Genymotion

发表于 2020-01-14 分类于 移动安全

Genymotion是一个跨平台的Android模拟器，支持Windows、Mac和Linux，基于x86架构，可以将Eclipse和Androidstudio完美整合到开发环境中。
下面介绍如何在deepin中安装和使用Genymotion。

Genymotion下载

Genymotion的中文官网是http://www.genymotion.net/，这个网站除了介绍Genymotion的特性之外并没有什么用，我们首先要注册账号：

正式的官网是https://www.genymotion.com/：
注册成功后登录官网，下载的页面是：https://www.genymotion.com/download/

除了在官网下载，还可以在deepin的深度商店上下载安装，但是在官网注册的步骤不能省略，后面配置需要登录。打开深度商店，找到“编程开发”，如下：

Genymotion安装

安装如下：

chmod 755 genymotion-2.11.0-linux_x64.bin
sudo mv genymotion-2.11.0-linux_x64.bin /opt
cd /opt
sudo ./genymotion-2.11.0-linux_x64.bin

配置Genymotion

启动Genymotin如下：

cd genymobile/genymotion/
./genymotion


启动后界面如下：

选择“Personal Use”即可打开主界面：

点击Add按钮添加虚拟机，要求登录：

登录后即可选择虚拟机版本，选择你需要的版本进行下载和安装：


安装完成后点击Start启动虚拟机：

安装APP

Genymotion是基于x86架构的，而Android APP是基于ARM架构的，两者之间并不兼容，所以如果直接安装APP会报错：

我们需要安装ARM的转换插件Genymotion-ARM-Translation。Genymotion-ARM-Translation可以直接百度下载，下载完成后直接拖放到虚拟机进行安装：


重启虚拟机之后即可正常安装APP。

使用adb

Genymotion默认使用的自带的adb，可以在Settings-ADB中进行设置：

不建议使用自定义adb，因为会和Genymotions自带的adb产生端口冲突。为了使用方便，我们可以定义别名。首先找到Genymotions自带的adb位置：

记下这个路径，编辑~/.bashrc文件：

sudo vim .bashrc

定义别名如下：

alias adb=/opt/genymobile/genymotion/tools/adb


然后执行以下指令应用更改：

source ~/.bashrc


无需连接端口即可直接使用adb shell：
